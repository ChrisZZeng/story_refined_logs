→ 边界：只使用已经呈现给玩家的信息，不用历史摘要补出玩家尚未看到的剧本后续。

→ 用途：理解本轮正文落实了什么，以及导演原本希望本轮正文停在哪里。
→ 边界：其中的约束主要约束正文生成，Choice 只用作边界参考；摘要和安排不能覆盖 current_text 的当前事实。

## current_text（本轮正文内容）
本轮正文内容。
→ 用途：选项生成的最终判断依据。选项必须从正文结尾的可见处境自然长出来。
→ 权重：最高。当前可行动处境以正文结尾为准；选项文案的具体事实优先来自这里。

<judgment_flow>
按这个顺序判断“玩家现在能点什么”，避免直接把后续剧本改写成选项。

1. 读 current_text 结尾：玩家此刻看到什么、知道什么、能做什么——这是唯一事实基础。
2. 参考 completed_plots、context 和 director_brief 确认当前节点已经推进到哪里。
3. 参考 plot_content / interactionFollowup 判断后续方向，但不要把玩家尚未看到的信息直接写成选项。
4. 最后匹配 candidate_actions，决定哪些选项需要绑定 actionId。
</judgment_flow>

<information_boundary>
控制选项文本里能出现什么信息，保证玩家不会通过选项提前看见隐藏内容或未来剧情。

- current_text 是当前可执行处境的最终判断依据。选项文案的具体事实优先来自 current_text；必要时可引用 completed_plots 或 context 中已经呈现给玩家的信息。
- script_guidance、entity_info、world_setting、candidate_actions 只提供方向和边界，不能把未呈现的信息写进选项文本。
- 如果正文还没完成空间转移（角色还在路上、门口、过渡中），不能生成目标空间内部的行动选项。
- 不要补充玩家没表达过的心理、动机、情绪或后续决定。
- 素材矛盾时取更保守的写法；不把 NPC 的猜测写成事实。
- 如果缺少执行某个行动的必要前提，应生成引导玩家靠近该场景的选项，而不是直接给出终点行动。
</information_boundary>

<action_binding>

- 多个不同选项可以绑定同一个 anchor:*，前提是它们是满足同一 anchor 的不同方式。

- 短且具体：每个选项像一个可点击的按钮文案——玩家一看就知道点了会触发什么
</option_quality>

## 文案质感
- 选项是玩家视角的主动行动，不是被动描述（"走进门" 不是 "你走进了门"）
- 点了之后会发生什么必须可预期——不要模糊到玩家不确定结果方向
- 措辞应贴合当前氛围和角色性格（紧张时简短急促，日常时可以稍松弛）
- 避免万金油选项：如"观察周围"这类在任何场景都成立的选项，除非当前情境确实需要观察

## 输出前自检
- 每个选项是否都能从 current_text 结尾的处境自然发生？
- 选项文本里的事实、名称、地点和关系，是否已在 current_text 或 context 中呈现给玩家？
- 是否把 script_guidance / candidate_actions 中尚未抵达的后续内容提前写成了选项？

- 选项之间是否互斥、具体，并且没有替玩家做重大决定？
