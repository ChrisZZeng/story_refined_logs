巷子的尽头。

摘要：帕兹跨过门槛进入卡琳娜真正的家，黑暗中被引导到内部空间。卡琳娜点亮烛火，家的陈设逐渐显现。卡尔从帆布包里跳出，以猫的形态观察帕兹。卡琳娜放松下来，邀请帕兹坐下，进一步透露她对敏特行踪的担忧以及康纳出现的隐患，暗示让帕兹暂时留在这里。卡尔对主角的态度出现微妙变化，从观察转向默许。

正文内容：
你跨过门槛。
黑暗在那一刻包裹了你——不是逐渐加深的那种，是你迈出那一步之后，身后的星光和巷子里的空气同时被切断，你整个人沉入了一个没有方向、没有边界、没有参考点的空间。你的视觉在进入后的第一个瞬间里几乎完全失效——你能感觉到自己的眼皮是睁开的，但你的视网膜接收到的信息为零。
你的右脚先落地。
鞋底触到的地面是硬的——不是石板或水泥，是木地板，表面有旧漆的质感，在你的体重压上去时没有发出声响，只有一种扎实的、被压实的触感传到你的脚底。你的左脚跟着跨进来时，鞋尖碰到了门槛的边缘——铁的触感通过鞋面传递——然后你的整个身体进入了这个空间。
你站在原地没有动。
黑暗是彻底的。你的目光在面前的方向中没有任何可以聚焦的点——没有光，没有轮廓，没有明暗交界，甚至连你自己的身体垂在身侧的手指，在你低头看时也只能感到它的存在，而看不到它的形状。
你听到她的声音。
来自你前方大约三步远的位置——不是从远处传来的那种距离感，是你在封闭空间中通过空气振动方向和声音的衰减程度能够判断出的那种具体距离。她的声音在说出第一个字时带着一个极轻微的定位感——她转了个身，面向你的方向。
“……别动。等我一下。”
她的声音在黑暗中比在有光的地方听起来更实。然后你听到她的脚步声——从刚才那个位置向你的右前方移动了大约两步——然后是布料摩擦的细响，然后是某种东西放在木质表面上的声音——可能是一张桌子或一个台面。然后是金属摩擦的声响——细微，连续——像是什么东西被从容器里拿出来。
擦的一声。
火柴。
那个声音在黑暗中格外清晰——火柴头划过摩擦面的瞬间，干燥的、磷质的声响——然后是一道橘黄色的光，短促地跳动了一下，然后稳定下来。
你的视网膜在那一瞬间恢复了工作。

你看到了她。
卡琳娜站在桌子旁，右手还握着火柴——火柴杆在她指间燃烧了大约一秒，她轻轻吹了一口气，火苗熄灭，一缕细烟从火柴末端升起，在烛火的暖黄色光中消散。她把熄灭的火柴放在桌面上一个敞开的铁盒边缘，然后转向你。

她开口了。
“这里就是我和卡尔住的地方。有点乱，但……很安全。”

包口边缘。
一双耳朵。
黑色的、三角形的猫耳朵——从包口内沿的边缘探出——先是左耳，然后是右耳。两耳之间的间距稳定，两只耳朵同时向内旋转了一个微小的角度——像是某个正在倾听的存在，在确认了外部环境的安全程度后，决定进行下一步动作。

落地的动作轻盈得几乎没有声响——四只爪子在木地板上先后触地，声音极轻，像几片干燥的树叶接连落到地面上。它在落地后没有立刻移动——四爪着地，尾巴从身体后方绕过来，尾尖在空中画出一个流畅的弧线——然后它坐了下来。

木地板在烛火的暖光中泛着柔和的暗光，它坐在那片光晕的中央——黑色短毛的猫，身体紧凑，四肢收拢在身下，尾巴从身体的右侧绕过前方，尾尖搭在右前爪旁边。它的瞳孔在烛火中保持着两道细窄的竖线——随着它头部的微小转动，那道竖线的宽窄和角度在缓慢地变化——但它始终看着你。

“坐吧。随便坐。”
她说这话时朝桌子和书架的方向示意了一下——那张木桌旁的椅子，以及壁炉前的地毯上可以垫坐的位置——然后她自己走向了房间另一侧，靠近壁炉的方向。她走到壁炉前，弯下腰——动作幅度不大，裙摆在弯腰时稍微向上提了一点——她伸手从壁炉旁一只藤编的柴篮里取出一根干柴，放在炉膛中已经架好的柴薪上。

壁炉里的火光在房间中增添了一层另一色彩的光——除了煤油灯的暖黄，还有一层更深的、跳动的橘红色火光。你的影子在墙壁上出现了两个方向——一个来自煤油灯，稳定清晰；一个来自壁炉，柔和而变幻不定。
卡琳娜在壁炉前蹲了一会儿。
她的目光落在跳动的火焰上——不是发呆，是一种她在看着某样东西的同时，正在进行的思考。她的右手无名指在她视线停留在火焰上时，在膝盖上轻轻敲了两下——节奏缓慢——然后停止了。

她的表情在壁炉的火光中显得比在外面时更松弛——她走向木桌旁的另一把椅子——之前你一直没有注意到那把椅子，它被放在书架的阴影中——她把它拉出来，自己坐下了。

环视一圈后说：‘这里很……实在。像有人真正住着。’

<output_requirements>
Director 本轮只输出结构化 JSON。

<output_schema>
只返回一个 JSON 对象，不写正文、对白、旁白或解释。

字段结构如下，字段名保持英文：
{
  “playerIntent”: string,
  “summary”: string,
  “involvedCharacters”: string[],
  “scene”: string,
  “beats”: string[],
  “characterBeats”: [{characterId, intent?, actionHint?}],
  “tone”: string,
  “pacing”: string,
  “requiredContent”: string[],
  “currentTurnConstraints”: string[],
  “currentStorylineConstraints”: string[],
  “sectionSignalSuggestion”: “continue”|”maybe_complete”|”complete”
}

可选字段没有内容时省略，不输出 null。
</output_schema>

<field_guide>
playerIntent（可选）——对玩家输入性质的判断：行动、探索、表达、对抗、初始开场。这是分类，不是剧情摘要。

summary——一句话概括本轮发生的剧情。供系统和 Narrator 快速把握全局。

involvedCharacters——本轮行动、说话，或需要被正文感知到在场/产生影响的角色 ID，使用稳定 ID。玩家主角若是本轮行动或观察的承载者，也可以作为参与角色列入，帮助 Narrator 把动作落在具体人物上。所有需要被正文感知到的角色，都应在 beats 或 characterBeats 中有可见位置。

scene（可选）——场景或位置发生变化时填写。

beats——本轮正文的剧情骨架，按顺序列出关键节拍。第一条 beat 是本轮正文的起点，最后一条是终点——Narrator 从第一条开始写，写完最后一条后停下。写事件和动作，不写文学描写。
  ✅ “帕兹向卡尔追问铃铛来历 → 卡尔用反问引导帕兹面对记忆空白 → 帕兹承认记忆分裂”
  ❌ “月光洒落石板路，脚步声在空旷巷子里回荡”

characterBeats——基于 beats 为重要角色提炼角色执行骨架：角色在本轮节拍中想达成什么、以怎样的动作、反应或语气推进。它帮助 Narrator 把 beats 落成具体角色表现，并补足角色层面的动作质感、潜台词和信息边界。
  - intent：角色本轮的意图或目标
  - actionHint：角色具体做什么，要具体到 Narrator 可执行。同时作为潜台词层——将隐藏信息翻译为可观察的行为质感，告诉 Narrator”怎么表现这个角色”而不是”角色背后是什么设定”
  ✅ actionHint: “沉默三秒后转移话题，语气刻意轻松”
  ❌ actionHint: “做出反应” / “她其实是眼线所以在套话”

tone（可选）——本轮情绪基调。给出明确方向。
  ✅ “表面平静但暗流涌动，对话中有试探和隐瞒”
  ❌ “适当的氛围”

pacing（可选）——给 Narrator 的呈现节奏建议。只写呈现方式，不写剧情原因。

requiredContent（可选）——本轮必须出现在正文中的固定演出：剧本设定的台词、固定剧情表演、引擎资源调用。
如果固定演出或固定对白有剧本顺序，必须先按顺序写入 beats 的对应位置；requiredContent 只记录不可漏、不可改义的具体内容，不单独承担顺序控制。

currentTurnConstraints（可选）——本轮 Narrator 需要遵守的约束集合。可以包含本轮新增的临时限制，也可以从 turn_constraints 中挑出本轮特别需要强调的条目。这是 Narrator 唯一能看到的约束来源。

currentStorylineConstraints（可选）——本轮新产生的、需要在后续持续遵守的约束。

sectionSignalSuggestion（可选）——当前故事线推进信号。continue：仍在推进中；maybe_complete：接近完成，本轮应有收束感；complete：本节点已完成。
</field_guide>

<output_discipline>
关于 requiredContent：
- 只放本轮必须出现在正文中的固定演出内容：剧本设定的台词、固定剧情表演、引擎资源调用（backgrounds、characters、cgs 等）
- 普通约束不往这里放

关于角色安排：
- beats 写清本轮事件顺序和推进边界；characterBeats 从这些事件中提炼重要角色的执行方式
- 主角行动若承载本轮重点，把主角也作为重要角色处理，写清他的行动意图、观察方式或身体反应。

关于固定剧情演出/固定对白的顺序：
- plot_content 中已经给出先后顺序的固定演出、固定对白，必须按原顺序消费，不得提前、延后、合并或跳过
- 每个固定演出或固定对白都应在 beats 中有对应位置，让 Narrator 按 beats 顺序落地
- requiredContent 用于保留固定对白原文、资源调用或不可改义的演出要求；它不能替代 beats 的顺序安排

关于 turn_constraints 的理解：
- turn_constraints 是辅助限制，不是情节安排的主要驱动。情节安排以 plot_content 和玩家输入为基准
- 如果 turn_constraints 中有明显与当前剧情阶段不再相关的内容，以当前故事节点的实际情况为准进行情节安排

关于 currentTurnConstraints 的输出：
- 这是 Narrator 唯一能看到的约束来源，你必须把本轮 Narrator 需要遵守的约束全部放在这里
- 从 turn_constraints 中选取本轮仍然有效且需要强调的条目，加上本轮新增的临时限制

关于 currentStorylineConstraints：
- 只在本轮确实产生了新的、后续需要持续遵守的限制时才输出
- 不得与已存在的约束语义重复
- 约束应具体可执行，不写模糊的情绪描述
</output_discipline>

<director_checklist>
输出前逐项确认：
1. 每个出场角色的行为，是否都有合理的信息来源？有无"不该知道却做出反应"的情况？
2. 本轮安排是否从 completed_plots 之后继续？是否和上轮使用了不同的情节结构？
3. 本轮推进量是否匹配故事节奏？对照 plot_content 剩余内容和已经过去的轮数，本轮安排的密度是否合理——不是只处理了一个微动作就停，也没有跳过必要的铺垫？
4. 若有固定剧情、固定演出或固定对白，是否已按剧本顺序完整消费，而非拆碎、重排或跳过？
5. 隐藏信息是否已翻译为行为/氛围，而非设定原文？
6. Narrator 必须遵守的约束，是否已经全部放入 currentTurnConstraints？
7. currentStorylineConstraints 是否和已有 constraints 语义不重复？
</director_checklist>
