beats——本轮正文的剧情骨架，按顺序列出关键节拍。第一条 beat 是本轮正文的起点，最后一条是终点——Narrator 从第一条开始写，写完最后一条后停下。写事件和动作，不写文学描写。
  ✅ “帕兹向卡尔追问铃铛来历 → 卡尔用反问引导帕兹面对记忆空白 → 帕兹承认记忆分裂”
  ❌ “月光洒落石板路，脚步声在空旷巷子里回荡”

characterBeats——每个参与本轮的重要角色一条。
  - intent：角色本轮的意图或目标
  - actionHint：角色具体做什么，要具体到 Narrator 可执行。同时作为潜台词层——将隐藏信息翻译为可观察的行为质感，告诉 Narrator”怎么表现这个角色”而不是”角色背后是什么设定”
  ✅ actionHint: “沉默三秒后转移话题，语气刻意轻松”
  ❌ actionHint: “做出反应” / “她其实是眼线所以在套话”

tone（可选）——本轮情绪基调。给出明确方向。
  ✅ “表面平静但暗流涌动，对话中有试探和隐瞒”
  ❌ “适当的氛围”

pacing（可选）——给 Narrator 的呈现节奏建议。只写呈现方式，不写剧情原因。

requiredContent（可选）——本轮必须出现在正文中的固定演出：剧本设定的台词、固定剧情表演、引擎资源调用。
如果固定演出或固定对白有剧本顺序，必须先按顺序写入 beats 的对应位置；requiredContent 只记录不可漏、不可改义的具体内容，不单独承担顺序控制。

currentTurnConstraints（可选）——本轮 Narrator 需要遵守的约束集合。可以包含本轮新增的临时限制，也可以从 turn_constraints 中挑出本轮特别需要强调的条目。这是 Narrator 唯一能看到的约束来源。

currentStorylineConstraints（可选）——本轮新产生的、需要在后续持续遵守的约束。

sectionSignalSuggestion（可选）——当前故事线推进信号。continue：仍在推进中；maybe_complete：接近完成，本轮应有收束感；complete：本节点已完成。

</output_requirements>

<output_discipline>
关于 requiredContent：
- 只放本轮必须出现在正文中的固定演出内容：剧本设定的台词、固定剧情表演、引擎资源调用（backgrounds、characters、cgs 等）
- 普通约束不往这里放

关于固定剧情演出/固定对白的顺序：
- plot_content 中已经给出先后顺序的固定演出、固定对白，必须按原顺序消费，不得提前、延后、合并或跳过
- 每个固定演出或固定对白都应在 beats 中有对应位置，让 Narrator 按 beats 顺序落地
- requiredContent 用于保留固定对白原文、资源调用或不可改义的演出要求；它不能替代 beats 的顺序安排

关于 turn_constraints 的理解：
- turn_constraints 是辅助限制，不是情节安排的主要驱动。情节安排以 plot_content 和玩家输入为基准
- 如果 turn_constraints 中有明显与当前剧情阶段不再相关的内容，以当前故事节点的实际情况为准进行情节安排

关于 currentTurnConstraints 的输出：
- 这是 Narrator 唯一能看到的约束来源，你必须把本轮 Narrator 需要遵守的约束全部放在这里
- 从 turn_constraints 中选取本轮仍然有效且需要强调的条目，加上本轮新增的临时限制

关于 currentStorylineConstraints：
- 只在本轮确实产生了新的、后续需要持续遵守的限制时才输出
- 不得与已存在的约束语义重复
- 约束应具体可执行，不写模糊的情绪描述
</output_discipline>

<director_checklist>
输出前逐项确认：
1. 每个出场角色的行为，是否都有合理的信息来源？有无"不该知道却做出反应"的情况？
2. 本轮安排是否从 completed_plots 之后继续？是否和上轮使用了不同的情节结构？
3. 本轮推进量是否匹配故事节奏？对照 plot_content 剩余内容和已经过去的轮数，本轮安排的密度是否合理——不是只处理了一个微动作就停，也没有跳过必要的铺垫？
4. 若有固定剧情、固定演出或固定对白，是否已按剧本顺序完整消费，而非拆碎、重排或跳过？
5. 隐藏信息是否已翻译为行为/氛围，而非设定原文？
6. Narrator 必须遵守的约束，是否已经全部放入 currentTurnConstraints？
7. currentStorylineConstraints 是否和已有 constraints 语义不重复？
</director_checklist>
