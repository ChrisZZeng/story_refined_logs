气窗正下方。你蹲下身，从旁边拖过一个木箱。箱子不大，边缘的木板被潮气浸得发软，但骨架还算结实。你把它叠到油桶顶上——先让一侧搭上桶面，再慢慢推到位。木板和桶身接触时发出轻微的刮擦声，你用手掌压住箱子，让它稳住。
你试了试高度。
脚尖踩上油桶边缘，体重压上去——桶身晃了一下，然后稳住。木箱在脚底微微位移，你调整了一下重心，让两只脚分别踩在木箱两侧，找平了受力点。
气窗的边缘就在你肩膀上方。
你伸手——指尖触到窗框的边缘。木质窗框表面粗糙，漆面剥落处露出灰褐色的木纹。你用手指沿着窗框摸了一圈，试探它的牢固程度。没有松动。窗扇的开合处有一个老式金属扣件，锈死了，但窗扇本身可以继续推开。
你深吸一口气。
手指扣住窗框边缘，慢慢往外推。窗扇在铰链上转动，发出一声极轻的吱呀——金属和木头摩擦的声音。你停住，等那声音完全消失，然后继续推。每推开一寸都停顿一下，让铰链适应新的角度。
缝隙足够大了。
你侧过身，肩膀先探进去。气窗开口宽度刚好让你侧身通过——胸贴着窗框边缘，背包从背上滑到一侧，你在狭窄的空间里调整角度，让背包先通过。布料的摩擦声在耳边响起，很轻，像粗布在木纹上擦过。
你把头探进内部。
昏暗。
比外部更暗——像是从黄昏走进了一个没有灯的储藏室。你眯起眼，让瞳孔适应。空气里有灰尘和旧纸箱的气味，还有某种化学品干涸后留下的味道。你的视野逐渐清晰——
一个储物间。
不算大。大概三四米见方。地面是水泥的，表面有一层灰褐色的积尘，混着碎纸片和干掉的污渍。墙壁没有粉刷，裸露的灰色砖体，有些地方钉着木板架。房间里堆着东西——
纸箱。很多纸箱。
沿着墙根堆叠，有些摞到天花板高度。箱体是统一的瓦楞纸板颜色，部分箱面有黑色的印刷标记——你认不出是哪家公司的包装。还有几块用帆布盖着的物体，轮廓在暗处显得模糊——有棱角，像机器或货箱。
没有人。
你扫视了整个房间。角落、纸箱之间的空隙、帆布覆盖物的阴影——没有看到有人活动的迹象。只有头顶一扇未开灯的灯泡，吊在天花板电线末端，灯泡表面落满灰，呈现出一种死寂的哑白色。
你侧身钻入。
膝盖先着地——右膝落在水泥地面上，发出一个沉闷的声响。你用手撑了一下地面，缓冲落地的冲击，然后把另一只脚也收进来。身体完全进入房间后，你蹲在原地，没有立刻起身。
你伸手，把气窗轻轻拉回原位。
窗扇闭合时铰链又发出一声轻响，你用手掌压住窗框边缘，直到它完全合拢。房间里恢复了原来的光线水平——只有门缝透进来的那一线昏黄。
你蹲在角落，没有动。
让眼睛继续适应这个光线。你扫视了一遍房间的布局——纸箱的堆叠方式，帆布的覆盖范围，地面上的脚印痕迹。在纸箱的间隙里，你看到一扇门——木质的，漆成深色，门缝处透出灯光。光线的颜色偏黄，带着那种老式白炽灯泡特有的暖调。
你压低身体，移动到最近的一个纸箱后面。
纸箱的高度刚好到你的腰部。你蹲在它后面，让箱体遮住你的轮廓。然后你贴着地面，朝那扇门移动——膝行，手掌撑地，让体重分布在三个支点上，尽量减少接触面积。水泥地面冰凉坚硬，膝盖压在上面传来微微的麻感。
你到了门边。
背靠着门边的墙体，蹲下。你先侧过头——耳朵贴住门扇，距离门缝一指宽。听。
脚步声。
隔着门板传来。不止一个人——脚步声在走廊的某个位置回响，鞋底踩在硬地面上的声音清晰，带着节奏。你数了一下——两个人的步频不同，一个稍快，一个更沉稳。然后是说话声——隔得太远，内容模糊不清，只能分辨出是两个不同的音色。
脚步声在靠近。
你调整了呼吸，没有后退。身体贴着墙面，让门框的边缘刚好遮住你的轮廓。光线从门缝里渗出来，在地面上画出一道狭长的亮痕，落在你的鞋尖前。
脚步声更近了。
然后经过——门缝的视野里，两个人影快速移动。走在前面的是一个人影，黑色上衣，步伐较快。后面一个人跟着，手里夹着一样东西——
公文包。
黑色的，皮革材质，在一个男人身侧夹着。他经过时公文包的一角在门缝视野的边缘晃动了一下，然后随着他的步伐远去。两个人消失在走廊尽头的方向——脚步声往更深处延伸，然后被一道门开合的声音截断。
走廊恢复安静。
你停在原地，维持着蹲姿，没有动。门缝里的灯光稳定地铺在地面上，灯丝偶尔闪一下，但光线没有变化。你等了三拍——没有脚步声折返，没有人说话。只有远处某个地方传来隐约的机器运转声，低沉的嗡嗡声，透过墙壁传递过来。
你慢慢抬起头。
目光从门缝移开，扫过你身后的储物间。
纸箱。帆布。架子。黑暗中堆叠的轮廓。你的视线在那些东西上逐一定格——屋顶有一根金属通风管，管道表面积着灰尘，接口处缠着银色的胶带，通向墙体上方的通风口。一根电线顺着墙角延伸，沿着踢脚线的位置，通往一个落满灰的配电箱。
你的目光落回最近的纸箱上。
边缘贴着透明胶带，已经被切割开过。你伸出手，用指尖轻轻掀开箱盖的一角——没有声音。箱口敞开，露出里面的东西：几件深色衣服，叠放整齐，上面压着一张纸。
你放下箱盖。
蹲在阴影里，手放在膝盖上。呼吸缓慢而稳定。门外那个走廊安静得像一条沉睡的通道。
</turn>
</context>

<output_requirements>
Director 本轮只输出结构化 JSON。

<output_schema>
只返回一个 JSON 对象，不写正文、对白、旁白或解释。

字段结构如下，字段名保持英文：
{
  “playerIntent”: string,
  “summary”: string,
  “involvedCharacters”: string[],
  “scene”: string,
  “beats”: string[],
  “characterBeats”: [{characterId, intent?, actionHint?}],
  “tone”: string,
  “pacing”: string,
  “requiredContent”: string[],
  “currentTurnConstraints”: string[],
  “currentStorylineConstraints”: string[],
  “sectionSignalSuggestion”: “continue”|”maybe_complete”|”complete”
}

可选字段没有内容时省略，不输出 null。
</output_schema>

<field_guide>
playerIntent（可选）——对玩家输入性质的判断：行动、探索、表达、对抗、初始开场。这是分类，不是剧情摘要。

summary——一句话概括本轮发生的剧情。供系统和 Narrator 快速把握全局。

involvedCharacters——本轮行动、说话，或需要被正文感知到在场/产生影响的角色 ID，使用稳定 ID。所有需要被正文感知到的角色，都应在 beats 或 characterBeats 中有可见位置。

scene（可选）——场景或位置发生变化时填写。

beats——本轮正文的剧情骨架，按顺序列出关键节拍。第一条 beat 是本轮正文的起点，最后一条是终点——Narrator 从第一条开始写，写完最后一条后停下。写事件和动作，不写文学描写。
  ✅ “帕兹向卡尔追问铃铛来历 → 卡尔用反问引导帕兹面对记忆空白 → 帕兹承认记忆分裂”
  ❌ “月光洒落石板路，脚步声在空旷巷子里回荡”

characterBeats——每个参与本轮的重要角色一条。
  - intent：角色本轮的意图或目标
  - actionHint：角色具体做什么，要具体到 Narrator 可执行。同时作为潜台词层——将隐藏信息翻译为可观察的行为质感，告诉 Narrator”怎么表现这个角色”而不是”角色背后是什么设定”
  ✅ actionHint: “沉默三秒后转移话题，语气刻意轻松”
  ❌ actionHint: “做出反应” / “她其实是眼线所以在套话”

tone（可选）——本轮情绪基调。给出明确方向。
  ✅ “表面平静但暗流涌动，对话中有试探和隐瞒”
  ❌ “适当的氛围”

pacing（可选）——给 Narrator 的呈现节奏建议。只写呈现方式，不写剧情原因。

requiredContent（可选）——本轮必须出现在正文中的固定演出：剧本设定的台词、固定剧情表演、引擎资源调用。
如果固定演出或固定对白有剧本顺序，必须先按顺序写入 beats 的对应位置；requiredContent 只记录不可漏、不可改义的具体内容，不单独承担顺序控制。

currentTurnConstraints（可选）——本轮 Narrator 需要遵守的约束集合。可以包含本轮新增的临时限制，也可以从 turn_constraints 中挑出本轮特别需要强调的条目。这是 Narrator 唯一能看到的约束来源。

currentStorylineConstraints（可选）——本轮新产生的、需要在后续持续遵守的约束。

sectionSignalSuggestion（可选）——当前故事线推进信号。continue：仍在推进中；maybe_complete：接近完成，本轮应有收束感；complete：本节点已完成。
</field_guide>
</output_requirements>

<output_discipline>
关于 requiredContent：
- 只放本轮必须出现在正文中的固定演出内容：剧本设定的台词、固定剧情表演、引擎资源调用（backgrounds、characters、cgs 等）
- 普通约束不往这里放

关于固定剧情演出/固定对白的顺序：
- plot_content 中已经给出先后顺序的固定演出、固定对白，必须按原顺序消费，不得提前、延后、合并或跳过
- 每个固定演出或固定对白都应在 beats 中有对应位置，让 Narrator 按 beats 顺序落地
- requiredContent 用于保留固定对白原文、资源调用或不可改义的演出要求；它不能替代 beats 的顺序安排

关于 turn_constraints 的理解：
- turn_constraints 是辅助限制，不是情节安排的主要驱动。情节安排以 plot_content 和玩家输入为基准
- 如果 turn_constraints 中有明显与当前剧情阶段不再相关的内容，以当前故事节点的实际情况为准进行情节安排

关于 currentTurnConstraints 的输出：
- 这是 Narrator 唯一能看到的约束来源，你必须把本轮 Narrator 需要遵守的约束全部放在这里
- 从 turn_constraints 中选取本轮仍然有效且需要强调的条目，加上本轮新增的临时限制

关于 currentStorylineConstraints：
- 只在本轮确实产生了新的、后续需要持续遵守的限制时才输出
- 不得与已存在的约束语义重复
- 约束应具体可执行，不写模糊的情绪描述
</output_discipline>

7. currentStorylineConstraints 是否和已有 constraints 语义不重复？
