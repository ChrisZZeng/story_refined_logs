脚步声更近了。
你听到碎石在他鞋底碾过的声音——比刚才更清晰。他已经走过空地的一半，方向依然带着那种漫不经心的弧度，但距离在缩短。你不需要看就能感觉到他的位置——在你的左前方，大概七八步远。
你保持不动。
膝盖弯着的角度没有变。背靠着墙的姿势没有变。目光低垂，落在木箱底部和地面之间的缝隙上——那里有一道细长的阴影，阳光从另一个方向斜过来，在地面上画出一道明暗分界线。你的视线就停在那条线上，没有移动。
脚步声停了。
不是那种走到目的地后自然停下的停法——是在一步迈出去之后，在半空中顿了一下，然后轻轻落地的停法。像是他看到了一样东西，需要停下来确认。
你的心脏在肋骨后面撞了一下。很重。你能感觉到它在胸腔里的震动，从内侧敲击着胸壁。然后第二下，轻了一点。第三下，恢复了节奏。
你没有眨眼。
你让目光保持低垂的姿态，看着地面那条阴影分界线。视网膜里有一些模糊的动静——边缘的余光捕捉到他的轮廓，站着，在你左前方几步远的位置。他停在那里，没有往前走，也没有后退。
你听到了呼吸声。
不是自己的。是另一个人的呼吸——很短，从鼻腔里出来，带着一种“嗯”的尾音。不是什么明确的表达，更像是脑子里闪过某个念头时下意识的发声。
然后安静了。
空气里只有风的声音。从斜后方灌进来的，吹动墙角堆着的编织袋，裂口处的碎布在风里轻轻抖动，布料摩擦的声音细碎而规律。远处有鸽子在叫，低沉的咕咕声，从一个屋顶传到另一个屋顶。
他还在看。
你知道他在看什么——你前面那个木箱。还有木箱旁边堆着的编织袋。还有墙角那些被雨水泡烂的纸箱碎片。他的目光大概在这些东西之间移动，缓慢的，像是要把每一个细节都看进眼里。
你没有动。
你的手指搭在相机上。拇指贴着快门旁边的金属片。食指自然弯曲，搭在机身侧面。你甚至能感觉到指腹下那一小块金属的温度——比空气凉一点，但比你的体温低不到哪里去。
时间在走。
大概过去了五秒。也可能更久。在你的感知里，那段沉默被拉得很长，像是有人把时间捻成丝，一根一根从纺锤上抽出来。
然后他动了。
不是朝你走。是他站在那里，把重心从左脚换到右脚——你能听到鞋底在碎石上碾动的声音。然后他转过身。不是突然的转身，是那种“没什么好看的”的转身，肩膀先转过去，然后是腰，然后是脚。
他开始往回走。
脚步声朝着空地的方向移动，每一步都在拉开距离。你听着那个声音——越来越轻，越来越远——直到在拐角处被墙体吞没。
你数了五下心跳。
然后你慢慢呼气。气息从喉咙里出来的时候带着一丝颤抖——很轻，像是被压了很久的弹簧终于松开了一点。你的肩膀往下沉了一点点，但身体没有改变蹲姿。手指还搭在相机上，没有移开。
你侧过头，从木箱和墙壁之间的缝隙看出去。
空地空了。
守门的站在铁门边，背靠着墙，双手插在口袋里。他侧着头，目光落在空地的另一个方向，和刚才搜索时的姿态完全不同——肩膀松着，站姿散漫，像是已经把刚才的事情翻过了页。
但你看到他右手插在口袋里的轮廓。
不是拳头松松地插着——是手指握着某个东西的形状。枪的轮廓，在布料下形成一道沉稳的突起。
他没有再往你的方向看。
你收回视线。
你缓慢地调整了一下蹲姿——膝盖从紧贴地面略微抬起来一点，让血液流回脚掌。麻木感像许多细针一样从脚底蔓延上来，你咬着牙，没有发出声音。
目光扫过你面前的木箱和编织袋。
位置不够好。你心里清楚。刚才那一眼不是偶然——他看到了这里有东西可以藏人。他没有查，只是决定不查。下一次也许不会。
你抬起头。
楼房侧面——和你这条巷子隔着大概两米宽的一道空隙。你之前没有注意到它，因为木箱和编织袋堆得太满，刚好挡住了那个方向的视线。但从你蹲着的位置侧过身，你能看到空隙的入口。
很窄。堆着杂物。几个倒地的自行车架，锈成一团废铁，轮子扭曲成奇怪的形状。还有几段钢管，几块碎木板，和一些认不出原貌的塑料制品——在昏暗的光线里堆成一堆，像被时间遗忘的垃圾。
但空隙通向后边。
通向楼房的后方。通向那扇铁门看不到的区域。
你的手指从相机上移开，搭在地面上。触到碎石和沙土的混合质感，凉的，粗糙的。你轻轻按压了一下地面，确认它足够坚实。
你看着那条堆满杂物的小巷，又看了一眼铁门边的守卫。
他没有在看你。

<output_schema>
只返回一个 JSON 对象，不写正文、对白、旁白或解释。

字段结构如下，字段名保持英文：
{
  “playerIntent”: string,

可选字段没有内容时省略，不输出 null。

<field_guide>
playerIntent（可选）——对玩家输入性质的判断：行动、探索、表达、对抗、初始开场。这是分类，不是剧情摘要。

summary——一句话概括本轮发生的剧情。供系统和 Narrator 快速把握全局。

involvedCharacters——本轮行动、说话，或需要被正文感知到在场/产生影响的角色 ID，使用稳定 ID。所有需要被正文感知到的角色，都应在 beats 或 characterBeats 中有可见位置。

scene（可选）——场景或位置发生变化时填写。

beats——本轮正文的剧情骨架，按顺序列出关键节拍。第一条 beat 是本轮正文的起点，最后一条是终点——Narrator 从第一条开始写，写完最后一条后停下。写事件和动作，不写文学描写。
  ✅ “帕兹向卡尔追问铃铛来历 → 卡尔用反问引导帕兹面对记忆空白 → 帕兹承认记忆分裂”
  ❌ “月光洒落石板路，脚步声在空旷巷子里回荡”

characterBeats——每个参与本轮的重要角色一条。

  - actionHint：角色具体做什么，要具体到 Narrator 可执行。同时作为潜台词层——将隐藏信息翻译为可观察的行为质感，告诉 Narrator”怎么表现这个角色”而不是”角色背后是什么设定”
  ✅ actionHint: “沉默三秒后转移话题，语气刻意轻松”
  ❌ actionHint: “做出反应” / “她其实是眼线所以在套话”

tone（可选）——本轮情绪基调。给出明确方向。
  ✅ “表面平静但暗流涌动，对话中有试探和隐瞒”
  ❌ “适当的氛围”

pacing（可选）——给 Narrator 的呈现节奏建议。只写呈现方式，不写剧情原因。

currentStorylineConstraints（可选）——本轮新产生的、需要在后续持续遵守的约束。

sectionSignalSuggestion（可选）——当前故事线推进信号。continue：仍在推进中；maybe_complete：接近完成，本轮应有收束感；complete：本节点已完成。

<output_discipline>
关于 requiredContent：
- 只放本轮必须出现在正文中的固定演出内容：剧本设定的台词、固定剧情表演、引擎资源调用（backgrounds、characters、cgs 等）
- 普通约束不往这里放

关于固定剧情演出/固定对白的顺序：
- plot_content 中已经给出先后顺序的固定演出、固定对白，必须按原顺序消费，不得提前、延后、合并或跳过
- 每个固定演出或固定对白都应在 beats 中有对应位置，让 Narrator 按 beats 顺序落地
- requiredContent 用于保留固定对白原文、资源调用或不可改义的演出要求；它不能替代 beats 的顺序安排

关于 turn_constraints 的理解：
- turn_constraints 是辅助限制，不是情节安排的主要驱动。情节安排以 plot_content 和玩家输入为基准
- 如果 turn_constraints 中有明显与当前剧情阶段不再相关的内容，以当前故事节点的实际情况为准进行情节安排

关于 currentTurnConstraints 的输出：
- 这是 Narrator 唯一能看到的约束来源，你必须把本轮 Narrator 需要遵守的约束全部放在这里
- 从 turn_constraints 中选取本轮仍然有效且需要强调的条目，加上本轮新增的临时限制

关于 currentStorylineConstraints：
- 只在本轮确实产生了新的、后续需要持续遵守的限制时才输出
- 不得与已存在的约束语义重复
- 约束应具体可执行，不写模糊的情绪描述
