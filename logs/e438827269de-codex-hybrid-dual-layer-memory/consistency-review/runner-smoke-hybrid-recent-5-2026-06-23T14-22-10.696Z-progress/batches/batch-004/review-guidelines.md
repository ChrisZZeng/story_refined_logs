# 本批次审阅规则

本批次使用 `batch_reviewer` skill 的规则：审阅对象是玩家可见经历，而不是隐藏剧本、内部状态或导演私有意图。

## 证据边界

- 可用证据：玩家输入、生成前已经发生且玩家可感知的事件、玩家实际看到的正文和选项。
- 不用证据：隐藏剧本事实、角色卡秘密、导演私有意图、内部运行时状态。
- 合理新增细节不是冲突；只有新增内容否定、改写或破坏此前玩家可见事实时，才计为问题。

## 批次规则

- 窗口范围提供上下文。
- 重点评估范围才计入本批次问题。
- 窗口上文中的问题不重复计入，除非它在重点评估范围内造成新的玩家可见问题。
- 每个问题必须标注 `scope`：`visibleText`、`choices`、`preLlmEvents` 或 `mixed`。

## 常用类型

- `fact-conflict`：人物、地点、物品、关系或事件前后矛盾。
- `identity-drift`：身份、称呼、性别、外貌或关系无解释漂移。
- `space-time-break`：地点、时间、行动顺序缺少必要承接。
- `user-input-ignored`：正文忽略、反驳或替换玩家输入。
- `event-negated`：此前玩家已经看到的事实在后文被说成没有发生或退回未发生状态。
- `unsupported-jump`：关键事件被写成已经发生，但此前玩家可见上下文没有支撑。
- `repeated-scene`：已经完整呈现过的情节、对白或行动被再次当作新内容演出。
- `language-drift`：正文语言、角色名、地点名或专有名词无理由切换。
- `protocol-break`：正文混入 JSON、markdown、系统说明、未解析标签等非正文内容。
- `quality-regression`：文本机械、残缺、无意义，或与互动小说上下文不相称。
- `other`：确实影响玩家可见体验但不适合归入以上类型。

## 严重程度

- `high`：直接破坏核心剧情理解、角色身份、地点关系、关键事件或语言可读性。
- `medium`：明显影响连续体验，但玩家仍能大致理解发生了什么。
- `low`：局部动作、物品、姿态或表达上的轻微冲突。

## 来源标注

- `player-input`：问题主要由玩家输入和此前可见上下文冲突造成。
- `model-output`：问题主要由模型输出造成。
- `mixed`：玩家输入带入冲突，模型输出又放大或固定冲突。
- `uncertain`：能确认体验有问题，但来源无法可靠判断。
