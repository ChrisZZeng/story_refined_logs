# 本批次审阅规则

本次审阅使用 `batch_reviewer` skill 及其 `references/review_guidelines.md`。审阅对象是玩家可见经历，不使用隐藏剧本、角色卡秘密、导演私有意图或内部运行状态作为判错依据。

## 证据范围

- 玩家已经输入的内容。
- 生成前已经发生、且应当影响玩家体验的 `preLlmEvents`。
- 玩家实际看到的 `visibleText` 和 `choices`。

## 统计范围

- 窗口范围：21-40，用作上下文和冲突证据。
- 重点评估范围：31-40，只有此范围内的问题计入本批次。

## Scope

- `visibleText`：问题发生在正文中。
- `choices`：问题发生在选项中。
- `preLlmEvents`：问题发生在生成前已发生事件中。
- `mixed`：问题横跨多个玩家可见部分。

## 判定重点

检查空间和时间连续性、行动和姿态连续性、物件和信息连续性、交互状态连续性、玩家意图承接、剧情进度连续性，以及语言、专名和格式稳定性。合理新增细节不算冲突；只有否定、改写或破坏此前玩家可见事实时才计为问题。明确的轻微连续性错误也记录为 `low`。

## 严重程度

- `high`：直接破坏核心剧情理解、角色身份、地点关系、关键事件或语言可读性。
- `medium`：明显影响连续体验，但玩家仍能大致理解发生了什么。
- `low`：局部动作、物品、姿态、表达或推进上的轻微冲突，值得记录但不视为严重失败。
