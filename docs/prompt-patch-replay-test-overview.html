测试关注的主链路

1. 配置校验
读取 replay config 和 patch bundle，拒绝不完整输入。
2. 版本锁
从日志和配置解析 oreturn commit，避免错版本 replay。
3. 上下文重建
从 run logs、review 结果、root cause 文件构建 turn context。
4. Prompt Patch
在同一 turn 调用链中精确应用 patch 并记录命中位置。
5. Replay/Judge/Report
准备 oreturn replay 输入，构造 judger 输入并生成汇总报告。

模块与测试功能

测试文件 对应模块 典型测试功能 核心风险
config.test.mjs config.mjs 校验 patch bundle、replay config、模型配置和默认版本策略。 输入格式错误导致 replay 半途失败，或配置字段被静默丢失。
source-version.test.mjs source-version.mjs 从日志目录名、run config、显式配置解析源 commit，并检测冲突。 使用了不同于 badcase 生成时的小说系统源码。
prompt-patcher.test.mjs prompt-patcher.mjs 精确替换 system/prompt/messages 内容，验证全调用链唯一命中。 patch 漏打、多打、跨 patch 互相制造命中或覆盖命中。
turn-context.test.mjs turn-context.mjs 从临时日志 fixture 构造单 turn replay context。 重建输入不完整，issue 定位错误，judger 上下文污染。
oreturn-engine.test.mjs oreturn-engine.mjs 生成 Bun eval 命令、写 replay-input、映射 LLM 环境变量。 调用不到 oreturn 模块，模型配置没有传入 replay 引擎。
judger.test.mjs judger.mjs 构造 judger 输入，校验 verdict 枚举，覆盖 fake 模式。 judge 输入缺旧输出/新输出/上下文，或产生不可识别 verdict。
report.test.mjs report.mjs 统计 verdict、失败 case、context-only issue，并渲染 Markdown。 summary 失真，patch 来源不可追溯，人工 review 找不到关键信息。

配置校验

test/prompt-patch-replay/config.test.mjs
4 cases
Case 输入 期望输出 意义
接受完整 patch bundle { id, patches: [{ id, originalText, replacementText }] } 通过，返回 bundle，保留 patch id。 保证用户提供的是“原始整段 + 替换整段”的精准 patch。
拒绝缺少 originalText { patches: [{ id, replacementText }] } 抛错，错误指向 patches[0].originalText。 避免没有原始段落时做模糊替换。

默认版本策略 source 未填写 versionPolicy。 默认补为 require-matching-worktree。 默认启用“必须匹配原 badcase 源码版本”的安全策略。

Prompt Patcher

test/prompt-patch-replay/prompt-patcher.test.mjs
10 cases
Case 输入 期望输出 意义
替换 message content messages[0].content = "before old paragraph after" 变为 "before new paragraph after"，记录 messages[0].content 和 sha256。 覆盖 Narrator/Choice 常见 messages 风格调用。
替换 system text system = "old paragraph"，prompt = "prompt stays" system 被替换，prompt 不变。 支持 System prompt 局部修复。
严格模式未命中失败 prompt = "no match" 抛错：not found。 用于单 call 严格 patch 校验。
同一 call 多次命中失败 prompt = "old paragraph old paragraph" 抛错：multiple matches。 迫使用户提供更长、更精准的原始段落。
当前 call 无 patch 时跳过 Director call 中没有该 patch 文本。 params 原样返回，applications=[]。 允许 patch 属于 Narrator 或 Choice，而不是每个子调用都必须命中。
跨调用链精确一次 Director 未命中，Narrator 命中一次。 Narrator 被替换，assertAllPatchesApplied 通过。 符合“一个 patch bundle 对一个 case 全链路至少且只能命中一次”。
整条调用链未应用失败 最终 applications=[]。 抛错：patch not applied。 保证 patch bundle 对每个 case 都真正生效。
已应用 patch 再出现失败 appliedPatchIds 已包含 p，后续 Choice 又出现原文。 抛错：already applied。 防止同一 patch 在同一 turn 调用链多次替换。
replacement 不能制造新命中 p1: alpha -> beta，p2: beta -> gamma，原文只有 alpha。 只应用 p1，最终 coverage 对 p2 失败。 patch 命中基于原始 assembled call，而不是前序 replacement 结果。
重叠 patch 失败 p1: alpha beta -> alpha，p2: beta -> gamma。 抛错：overlap/removed。 防止 patch A 覆盖掉 patch B 的原始命中。

Case 输入 期望输出 意义
统计 verdict 和失败 case 一个 completed case 有 fixed、improved，一个 failed case 错误为 patch not found。 fixed=1、improved=1、failedCases=1，保留 patch path/hash。 summary 能表达本次 patch bundle 的修复效果和失败范围。

当前测试边界

当前测试覆盖的是第一版 replay 系统的核心纯逻辑和入口约束：配置、版本锁、patch 命中、context 重建、engine 输入、judge 结构和 report 汇总。

它没有在单元测试里真实调用外部 LLM，也没有端到端启动完整小说生成。真实 replay 的额外保障来自 CLI smoke：--dry-run-context-only 可验证日志重建，非 dry-run 在 oreturn commit 不匹配时会拒绝执行。

建议后续增加一组可控 fake LLM 的集成测试，用固定 oreturn fixture 验证 Director -> Narrator -> Choice 的完整调用链落盘。
